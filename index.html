Concentrate

A card game created in MIPS assembly language

In CS 0447 Computer Organization & Assembly Language during the Spring of my first year, I recreated the card game 'Concentrate' in MIPS assembly language. This was done through an LED board extension to MIPS my professor made for the course.

The objective of the game is to match all cards in three minutes. It is played by flipping cards over in pairs to see if they match. A mismatch will cause a three-second delay. A match will simply continue the game. Matching a smiley faced card will extend the timer by 30 seconds; however, the smiley card changes every 30 seconds so the player must be quick.

Word Finder

A word finder using backtracking in Java

In CS 0445 Data Structures, I created this Java program that reads a word grid of any rectangular size from a .txt file to determine whether a word or phrase, which the user is prompted to enter, exists in the grid. If found, the program outputs the grid with the word or phrase in uppercase text.

The algorithm basically parses the entire grid in a 2D array and recursively searches for the word or phrase. The direction can only change at the end of each word so if the next letter is not found in the same direction, the program stops search and moves on in the grid.



There were other programs I made in this class but this was the most fun and interesting to create by far.

  • Category: School Project

Kobuki Robot

Project-oriented robotics class

In my Programming System Design on a Mobile Robot Platform class during the fall of my second year, I worked in a team of two on various interesting projects with the Kobuki robot. The class was composed of only projects so for the whole semester, my partner and I worked extensively in Python to make the robot do cool things.

Project 1 was called Sennott Square Rover and was fairly simple. My partner and I were required to be able to SSH to the robot a single command or a series of commands for movement (ie. Forward 2 meters, Right 90°, ...). We had to account for wheel slippage by creating an algorithm that would gradually accelerate the robot for any distance input.

In Project 2, Soccer Kick, the task was to use the webcam of the laptop mounted on the robot to follow a colored line on the ground and to make sure that during a sharp turn, the robot would not go off of the line. Then, the robot had to scan the environment through the webcam for a red ball and then an orange goal. Using only three linear movements, the robot had to kick the ball into the goal.

Project 3 was called Selfie Bot and it incorporated a Microsoft Kinect. The robot was to follow a person around the building automatically using only colored construction paper on cardboard. Using the Kinect, the robot recognized the color pattern on the cardboard and also the distance to the person to decide whether to move forward, backward, or turn and how fast. On the other side of the cardboard, there was another pattern. If the robot saw this pattern, it would start a countdown and at 0, it would take a selfie of the user.

Project 4 was called DJ Roomba and the idea for it was mine, inspired by the TV show Parks and Recreation. Essentially, the robot was to move randomly throughout the building and play music. For the class, this was too simple so I decided to incorporate voice recognition. Using the Google Speech Recognition API and the Spotify Web API, my partner and I recreated DJ Roomba with a modern twist. We could say "Mr. Data" (the robots name) and "play artist ..." and the robot would stop, recognize the command, play a random song from the top 20 results of the artist on Spotify, and then continue moving about. Another command was playing a specific song by an artist.
